# Test Certificates for EZ Enforcer mTLS

This folder contains keys and certificates for testing mTLS integration in EZ enforcer, used to
secure communications between Encrypted Zones (EZs).

## Certificate Roles

-   **Leaf Certificate (`leaf.*`)**: Represents the certificate generated by an EZ. The EZ enforcer
    is expected to load `leaf.key` as its private key and use `leaf.crt` for the certificate signing
    request (CSR) sent to the `EzMtlsService.GetCertificates` RPC.
-   **Root Certificate (`root.*`)**: Represents the certificate generated by TCA / AVS. The
    combination of `leaf.der` and `root.der` will be sent back to and trusted by EZ.

## Certificate Generation

The certificates are generated using `openssl` with the following parameters:

### Root Certificate

```shell
openssl genrsa -out root.key 2048
openssl req -x509 -new -nodes -key root.key -sha256 -days 3650 -out root.crt \
    -subj "/C=US/ST=WA/L=Seattle/O=Google/OU=PrivacySandbox" \
    -addext "subjectAltName = DNS:avs.tca.fakeca"
```

### Leaf Certificate

```shell
openssl genrsa -out leaf.key 2048
openssl req -new -key leaf.key -out leaf.csr \
    -subj "/C=US/ST=WA/L=Seattle/O=Google/OU=PrivacySandbox" \
    -addext "subjectAltName = URI:spiffe://avs.tca.fakeca/operator/privacy-sandbox-dev-jobs@prod.google.com/publisher/release@google.com/workload/encrypted-zone/v/1.0.0, DNS:*.avs.tca.fakeca"
```
